If you use Spec2Vec in your research, please cite both the software repository and the accompanying academic paper. Proper citation supports continued development and ensures reproducibility across the geophysical and broader signal-processing communities.

## Paper Citation (In Review)

The following BibTeX entry covers the primary academic paper describing the Spec2Vec methodology, its information-theoretic foundations, and its applications to planetary signal characterization.

```bibtex
@article{Swar2026spec2vec,
  author  = {Swar, Sayan and Mittal, Tushar and Olugboji, Tolulope},
  title   = {{Spec2VEC}: Interpretable Spectral Representations of Planetary Signals in Vector Spaces},
  journal = {Geophysical Journal International},
  year    = {2026},
  note    = {In review}
}
```

<Note>
  This paper is currently in review at *Geophysical Journal International*. Please check the [repository](https://github.com/URseismology/Spec2Vec) for the final published citation once the paper is accepted and assigned a DOI.
</Note>

## Software Repository Citation

If you use the code directly — whether you import `GISQAPipeline`, adapt the notebooks, or build on the utility functions — please also cite the software repository itself.

```bibtex
@software{Swar2026spec2vec,
  author  = {Swar, Sayan and Mittal, Tushar and Olugboji, Tolulope},
  title   = {Spec2Vec: Interpretable Spectral Representations of Planetary Signals in Vector Spaces},
  year    = {2026},
  url     = {https://github.com/URseismology/Spec2Vec},
  note    = {GitHub repository}
}
```
