`GISQAPipeline` is the main entry point to Spec2Vec. It is a high-level class that wraps both pointwise and pairwise feature extraction into a clean API, letting you go from a list of raw arrays to a labelled pandas DataFrame in a single method call. You instantiate it once and then call whichever extraction method fits your workflow — there is no configuration file to edit and no subclassing required.

```python
from SPEC2VEC.utils.gisqa_compute_updated import GISQAPipeline

giq = GISQAPipeline()
```

---

## Pointwise pipeline

The pointwise pipeline extracts scalar features from each signal or spectrogram independently, with no reference to other signals. It is fast, self-contained, and requires no external data beyond the input arrays themselves.

### Method signature

```python
df = giq.compute_pointwise_metrics_from_spec(
    spec_names,
    spec_images,
    is_hilbertize=1,
    is_normalize_stat=0,
    is_normalize_entropy=0,
    is_best_features=0,
    ordpydx=5,
    antropydx=5,
    metrics_list=None,
    hilbert_locs=None
)
```

### Parameters

<ParamField path="spec_names" type="list[str]" required>
  A list of string identifiers for each signal. These become the values in the `label` column of the returned DataFrame and are used to track which row corresponds to which input. The list must have the same length as `spec_images`.
</ParamField>

<ParamField path="spec_images" type="list[np.ndarray]" required>
  A list of NumPy arrays to process. Each element can be a **1D array** (raw time-series) or a **2D array** (spectrogram or scalogram). All arrays in the list may have different shapes — the pipeline processes them one at a time.
</ParamField>

<ParamField path="is_hilbertize" type="int" default="1">
  Set to `1` to enable Hilbert curve flattening for 2D spectrogram inputs. The pipeline automatically selects the standard Hilbert curve for square power-of-2 images and the generalised Gilbertised curve for rectangular images. Set to `0` when passing 1D time-series arrays directly.
</ParamField>

<ParamField path="is_normalize_stat" type="int" default="0">
  Set to `1` to normalise the flattened signal to the range `[-1, 1]` before computing **statistical features** (mean, variance, kurtosis, etc.). Normalisation is performed with `MinMaxScaler`. Normalising statistical features can improve comparability across signals with different amplitude scales.
</ParamField>

<ParamField path="is_normalize_entropy" type="int" default="0">
  Set to `1` to normalise the flattened signal before computing **entropy and fractal features** (permutation entropy, DFA, etc.). Note that some entropy measures — particularly `svd_entropy_antropy` — are sensitive to the amplitude scaling of the input, so evaluate whether normalisation helps or hurts for your specific dataset.
</ParamField>

<ParamField path="is_best_features" type="int" default="0">
  Set to `1` to return only the five most discriminating features identified across geophysical datasets: `detrended_fluctuation_antropy`, `fisher_shannon_ordpy`, `Lempel-Ziv complexity`, `svd_entropy_antropy`, and `approximate_entropy_antropy`. When `metrics_list` is also provided, `is_best_features` is overridden by `metrics_list`.
</ParamField>

<ParamField path="ordpydx" type="int" default="5">
  The embedding dimension (`dx`) passed to all `ordpy` feature computations. Controls the length of ordinal patterns evaluated (pattern length = `ordpydx`). Recommended range: **4–6**. Higher values capture longer-range correlations but increase computation time and require longer input signals.
</ParamField>

<ParamField path="antropydx" type="int" default="5">
  The embedding dimension (`dx`) passed to all `antropy` entropy and fractal computations. Affects permutation entropy, SVD entropy, approximate entropy, and sample entropy. Recommended range: **4–6**. Must be consistent across your dataset for results to be comparable.
</ParamField>

<ParamField path="metrics_list" type="list[str] | None" default="None">
  An explicit list of feature names to compute. When provided, only the listed features are extracted — all others are skipped, which substantially speeds up processing on large datasets. Feature names must use the canonical suffixed form: `_antropy`, `_ordpy`, or the plain TSFEL name (e.g., `'Lempel-Ziv complexity'`). When `None`, all features are computed (unless `is_best_features=1`).
</ParamField>

<ParamField path="hilbert_locs" type="np.ndarray | None" default="None">
  Pre-computed Hilbert curve index array of shape `(N_pixels, 2)`, where each row is a `(row, col)` coordinate. If you are processing a large batch of spectrograms that all have the same shape, pass a pre-computed index here to avoid re-running the curve generation on every call. Generate it once with `HelperFunc.gilbertize_image_optimized(width=H, height=W)`.
</ParamField>

### Full worked example

```python
import numpy as np
from SPEC2VEC.utils.gisqa_compute_updated import GISQAPipeline
from SPEC2VEC.utils.spectograms_lib import stft_basic_spectogram

# --- Generate two synthetic signals ---
sr = 100      # Sampling rate in Hz
t_len = 5     # Duration in seconds
x = np.linspace(0, t_len, sr * t_len)

y1 = np.sin(2 * np.pi * 35 * x) * np.exp(-((x - 4) ** 2) / (2 * 0.3 ** 2))
y2 = np.sin(2 * np.pi * 20 * x) * np.exp(-((x - 1) ** 2) / (2 * 0.3 ** 2))

# --- Compute STFT spectrograms ---
f, t_arr, stft1 = stft_basic_spectogram(
    y1, sr, nperseg=64, overlap=0.5, window='hann',
    f_min=0, f_max=50,
    max_normalize=False, powerlog=False,
    vmin_percentile=2, vmax_percentile=98
)
_, _, stft2 = stft_basic_spectogram(
    y2, sr, nperseg=64, overlap=0.5, window='hann',
    f_min=0, f_max=50,
    max_normalize=False, powerlog=False,
    vmin_percentile=2, vmax_percentile=98
)

# --- Extract pointwise features ---
giq = GISQAPipeline()
df = giq.compute_pointwise_metrics_from_spec(
    ['signal_35hz', 'signal_20hz'],
    [stft1, stft2],
    is_hilbertize=1,       # Enable Hilbert curve for 2D input
    is_normalize_stat=0,
    is_normalize_entropy=0,
    is_best_features=0,    # Compute all features
    ordpydx=5,
    antropydx=5,
    metrics_list=None,
    hilbert_locs=None
)

print(df.shape)            # (2, N_features + 1)
print(df.columns.tolist()) # All feature names plus 'label'
```

## Pairwise pipeline

The pairwise pipeline, accessed through `execute_pairwise_pipeline()`, compares a target spectrogram against a pre-built **noise reference bank** to extract spatial zone metrics. It is more computationally expensive than the pointwise pipeline but provides richer discriminative features for noisy real-world data.

### The three sub-steps

<Steps>
  <Step title="Histogram alignment">
    The amplitude distribution of the target spectrogram is compared against every entry in the noise bank using cross-correlation and differential evolution optimisation. The noise entry whose aligned distribution best matches the target is selected as the reference. The optimal scaling factor stretches the noise distribution to overlap with the target, effectively isolating the signal of interest from background noise.
  </Step>
  <Step title="GMM labeling">
    A Gaussian Mixture Model (GMM) is fitted to the residual distribution — the portion of the spectrogram that exceeds the noise threshold. The GMM components define cut thresholds along the intensity axis, and every pixel in the spectrogram is assigned an integer energy label (0 = noise floor, 1 = low signal, 2 = moderate signal, and so on).
  </Step>
  <Step title="GLSZM spatial zone metrics">
    The labeled spectrogram is analysed using a Gray-Level Size Zone Matrix (GLSZM) from the `pyradiomics` library. This quantifies how energy patches are spatially distributed — whether high-energy pixels form large coherent zones or are scattered as small isolated spots. Outputs include Small Area Emphasis, Large Area Emphasis, Gray Level Non-Uniformity Normalized, Size Zone Non-Uniformity Normalized, Large Area High Gray Level Emphasis, and a custom Weighted Zone Size Percentage. A second-pass refinement removes small speckle zones before recomputing, yielding `_stg1` and `_stg2` variants of every metric.
  </Step>
</Steps>

### When to use pairwise vs pointwise

| Consideration | Pointwise | Pairwise |
|---|---|---|
| Requires noise bank | No | Yes (HDF5 file) |
| Computation time | Fast | Slow (optimisation loop) |
| Works on 1D time-series | Yes | No (needs 2D spectrogram) |
| Captures spatial structure | Indirectly | Explicitly |
| Best for | Exploration, large batches | Production, noisy real data |

<Warning>
  The pairwise pipeline requires a pre-built noise bank in HDF5 format. You can generate one using `GISQAPipeline.compute_imgbank_dict()` or `compute_noisebnk_metrics()`. Without it, the histogram alignment step has no reference to align against.
</Warning>

## Normalization options

Two independent normalisation flags let you control whether the flattened signal trace is rescaled before metric computation.

<CardGroup cols={2}>
  <Card title="is_normalize_stat" icon="ruler">
    Scales the trace to `[-1, 1]` before computing **statistical features** (mean, variance, RMS, percentiles, etc.). Use this when your signals have very different amplitude ranges and you want the statistical features to reflect distributional shape rather than absolute scale.
  </Card>
  <Card title="is_normalize_entropy" icon="lock">
    Scales the trace to `[-1, 1]` before computing **entropy and fractal features**. Most entropy measures are already normalised by construction (permutation entropy returns values in [0,1]), but amplitude-sensitive measures like `svd_entropy_antropy` may benefit. Test on your data before enabling.
  </Card>
</CardGroup>

Both flags default to `0` (disabled). You can enable them independently — for example, `is_normalize_stat=1, is_normalize_entropy=0` normalises only the statistical branch.

## Feature selection

Spec2Vec offers two complementary ways to control which features are returned, letting you trade completeness for speed.

### Using `is_best_features`

Setting `is_best_features=1` returns exactly five features that have been empirically validated as the top discriminators across multiple geophysical signal datasets:

```python
df = giq.compute_pointwise_metrics_from_spec(
    spec_names, spec_images,
    is_hilbertize=1,
    is_best_features=1,     # Return only the top 5
    ordpydx=5, antropydx=5,
    is_normalize_stat=0, is_normalize_entropy=0,
    metrics_list=None, hilbert_locs=None
)
# df will have 6 columns: the 5 best features + 'label'
```

### Using `metrics_list`

For fine-grained control, pass an explicit list of feature names. Only those features — and the computations required to produce them — will be executed. This is the recommended approach for production pipelines:

```python
metrics_to_compute = [
    'detrended_fluctuation_antropy',
    'svd_entropy_antropy',
    'fisher_shannon_ordpy',
    'Lempel-Ziv complexity',
    'Kurtosis',
    'Root mean square',
]

df = giq.compute_pointwise_metrics_from_spec(
    spec_names, spec_images,
    is_hilbertize=1,
    metrics_list=metrics_to_compute,
    ordpydx=5, antropydx=5,
    is_normalize_stat=0, is_normalize_entropy=0,
    is_best_features=0, hilbert_locs=None
)
```

<Tip>
  When `metrics_list` is provided, `is_best_features` is ignored. The pipeline computes only what you asked for and returns those columns plus `label`.
</Tip>

## Complete end-to-end example

The example below walks through signal generation, spectrogram computation, and pointwise feature extraction in a single script — the minimal reproducible workflow for Spec2Vec.

```python
import numpy as np
from SPEC2VEC.utils.gisqa_compute_updated import GISQAPipeline
from SPEC2VEC.utils.spectograms_lib import stft_basic_spectogram

# --- Synthetic signal: Gaussian-windowed sine burst ---
sr = 100
t = 5
x = np.linspace(0, t, sr * t)
y = np.sin(2 * np.pi * 35 * x) * np.exp(-((x - 4) ** 2) / (2 * 0.3 ** 2))

# --- Compute spectrogram ---
f, t_arr, stft = stft_basic_spectogram(
    y, sr, 64, 0.5, 'hann', 0, 50,
    max_normalize=False, powerlog=False,
    vmin_percentile=2, vmax_percentile=98
)

# --- Extract features ---
giq = GISQAPipeline()
df = giq.compute_pointwise_metrics_from_spec(
    ['my_signal'], [stft],
    is_hilbertize=1,
    is_normalize_stat=0,
    is_normalize_entropy=0,
    is_best_features=0,
    ordpydx=5,
    antropydx=5,
    metrics_list=None,
    hilbert_locs=None
)

print(df.shape)            # (1, N_features)
print(df.columns.tolist()) # List of feature names
```
